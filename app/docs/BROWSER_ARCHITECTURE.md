# 浏览器架构：从零到一的开发过程

## 核心概念

```
浏览器 = 多个引擎 + 多个组件的集合

主要引擎：
├─ 渲染引擎（Rendering Engine）
├─ JavaScript 引擎（JavaScript Engine）
├─ 网络引擎（Network Engine）
└─ 存储引擎（Storage Engine）

主要组件：
├─ UI 层
├─ 进程管理
├─ 缓存系统
├─ 安全系统
└─ 等等
```

## 第二部分：渲染引擎（Rendering Engine）

### 什么是渲染引擎

```
渲染引擎 = 将 HTML/CSS 转换为像素的引擎

输入：HTML、CSS、JavaScript
处理：解析、计算、绘制
输出：像素、显示在屏幕上
```

### 主要的渲染引擎

**Blink（Chrome、Edge、Opera）**
```
开发者：Google
基于：WebKit
特点：
├─ 高性能
├─ 现代化
├─ 开源
└─ 广泛使用

代码量：~200 万行
```

**Gecko（Firefox）**
```
开发者：Mozilla
特点：
├─ 高性能
├─ 开源
├─ 独立开发
└─ 注重标准

代码量：~200 万行
```

**WebKit（Safari）**
```
开发者：Apple
特点：
├─ 高性能
├─ 开源
├─ 针对 Apple 优化
└─ 注重隐私

代码量：~150 万行
```

### 渲染引擎的工作流程

```
1. 解析 HTML
   ├─ 词法分析
   ├─ 语法分析
   └─ 生成 DOM 树

2. 解析 CSS
   ├─ 词法分析
   ├─ 语法分析
   └─ 生成 CSSOM 树

3. 合并 DOM 和 CSSOM
   └─ 生成渲染树

4. 布局（Layout）
   ├─ 计算每个元素的位置
   ├─ 计算每个元素的大小
   └─ 生成布局树

5. 绘制（Paint）
   ├─ 将布局树转换为绘制指令
   ├─ 生成绘制列表
   └─ 分层处理

6. 合成（Composite）
   ├─ 将绘制列表转换为位图
   ├─ 上传到 GPU
   └─ 显示在屏幕上
```

### 渲染引擎的核心算法

**HTML 解析**
```
HTML 文本
    ↓
词法分析（Tokenization）
    ↓
生成 Token 流
    ↓
语法分析（Parsing）
    ↓
生成 DOM 树
```

**CSS 解析**
```
CSS 文本
    ↓
词法分析
    ↓
生成 Token 流
    ↓
语法分析
    ↓
生成 CSSOM 树
```

**布局计算**
```
渲染树
    ↓
遍历每个节点
    ↓
计算位置和大小
    ↓
考虑 CSS 属性
    ↓
生成布局树
```

**绘制**
```
布局树
    ↓
遍历每个节点
    ↓
生成绘制指令
    ↓
分层处理
    ↓
生成绘制列表
```

## 第三部分：JavaScript 引擎

### 什么是 JavaScript 引擎

```
JavaScript 引擎 = 执行 JavaScript 代码的引擎

输入：JavaScript 代码
处理：解析、编译、执行
输出：执行结果
```

### 主要的 JavaScript 引擎

**V8（Chrome、Node.js）**
```
开发者：Google
特点：
├─ 高性能
├─ JIT 编译
├─ 垃圾回收
├─ 开源
└─ 广泛使用

代码量：~100 万行
```

**SpiderMonkey（Firefox）**
```
开发者：Mozilla
特点：
├─ 高性能
├─ JIT 编译
├─ 垃圾回收
├─ 开源
└─ 注重标准
```

**JavaScriptCore（Safari）**
```
开发者：Apple
特点：
├─ 高性能

├─ 垃圾回收
├─ 开源
└─ 针对 Apple 优化

代码量：~50 万行
```

### JavaScript 引擎的工作流程

```
1. 解析 JavaScript
   ├─ 词法分析
   ├─ 语法分析
   └─ 生成 AST（抽象语法树）

2. 编译
   ├─ 解释执行（Interpreter）
   ├─ JIT 编译（Just-In-Time Compilation）
   └─ 优化

3. 执行
   ├─ 创建执行上下文
   ├─ 执行代码
   ├─ 管理内存
   └─ 垃圾回收

4. 与 DOM 交互
   ├─ 修改 DOM
   ├─ 触发重排
   ├─ 触发重绘
   └─ 更新页面
```

### V8 引擎的详细工作流程

**解析阶段**
```
JavaScript 代码
    ↓
词法分析（Lexical Analysis）
    ├─ 将代码分解为 Token
    └─ 例如：var、=、x、1、;
    ↓
语法分析（Syntax Analysis）
    ├─ 检查语法是否正确
    └─ 生成 AST
    ↓
AST（抽象语法树）
```

**编译阶段**
```
AST
    ↓
解释执行（Interpreter）
    ├─ 快速启动
    ├─ 代码立即执行
    └─ 性能一般
    ↓
JIT 编译（Just-In-Time Compilation）
    ├─ 监测热点代码
    ├─ 编译为机器码
    ├─ 性能很好
    └─ 启动慢
    ↓
优化
    ├─ 内联优化
    ├─ 消除死代码
    ├─ 常数折叠
    └─ 等等
```

**执行阶段**
```
编译后的代码
    ↓
创建执行上下文
    ├─ 全局上下文
    ├─ 函数上下文
    └─ 块级上下文
    ↓
执行代码
    ├─ 变量赋值
    ├─ 函数调用
    ├─ 对象创建
    └─ 等等
    ↓
内存管理
    ├─ 分配内存
    ├─ 追踪引用
    └─ 垃圾回收
```

### V8 的垃圾回收

```
V8 使用分代垃圾回收

新生代（Young Generation）：
├─ 存储新创建的对象
├─ 频繁进行垃圾回收
├─ 回收速度快
└─ 大多数对象在这里被回收

老生代（Old Generation）：
├─ 存储长期存活的对象
├─ 不频繁进行垃圾回收
├─ 回收速度慢
└─ 对象在这里长期存活

垃圾回收算法：
├─ Scavenge（新生代）
├─ Mark-Sweep（老生代）
├─ Mark-Compact（老生代）
└─ Incremental Marking（增量标记）
```

## 第四部分：网络引擎

### 什么是网络引擎

```
网络引擎 = 处理网络请求的引擎

功能：
├─ DNS 解析
├─ TCP 连接
├─ HTTP 请求
├─ 缓存管理
├─ Cookie 管理
├─ HTTPS 支持
└─ 等等
```

### 网络引擎的工作流程

```
1. DNS 解析
   ├─ 查询本地缓存
   ├─ 查询系统 DNS
   ├─ 查询 ISP DNS
   └─ 获得 IP 地址

2. TCP 连接
   ├─ 三次握手
   ├─ 建立连接
   └─ 准备发送数据

3. HTTP 请求
   ├─ 构建请求头
   ├─ 发送请求
   ├─ 接收响应
   └─ 处理响应

4. 缓存管理
   ├─ 检查缓存
   ├─ 验证缓存
   ├─ 更新缓存
   └─ 清理缓存

5. 关闭连接
   ├─ 四次挥手
   ├─ 关闭连接
   └─ 释放资源
```

### 网络引擎的优化

**连接复用**
```
HTTP/1.1：持久连接
├─ 一个连接可以发送多个请求
├─ 减少连接建立时间
└─ 提高性能

HTTP/2：多路复用
├─ 一个连接可以并行发送多个请求
├─ 进一步提高性能
└─ 减少延迟

HTTP/3：QUIC 协议
├─ 基于 UDP
├─ 更快的连接建立
├─ 更好的性能
└─ 无队头阻塞
```

**缓存管理**
```
浏览器缓存：
├─ 内存缓存
├─ 磁盘缓存
├─ Service Worker 缓存
└─ 等等

缓存策略：
├─ 强缓存（不发送请求）
├─ 协商缓存（发送请求，检查是否更新）
└─ 无缓存（总是发送请求）
```
